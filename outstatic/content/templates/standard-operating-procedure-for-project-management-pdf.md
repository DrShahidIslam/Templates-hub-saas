# Standard Operating Procedure: Project Management Documentation & PDF Archiving

This Standard Operating Procedure (SOP) outlines the mandatory workflow for creating, formatting, and finalizing project management documentation into Portable Document Format (PDF). Adherence to these standards ensures consistency across project lifecycles, maintains document integrity, and facilitates efficient retrieval for stakeholders and auditors. By following this protocol, project teams minimize version control risks and maintain a professional repository of all critical project artifacts.

## Phase 1: Preparation and Drafting
*   **Establish Document Template:** Utilize the organization’s approved project management document template (e.g., Project Charter, Status Report, or Risk Log).
*   **Version Control:** Include a version control table on the title page or header, noting the author, date, version number (e.g., v1.0), and a brief description of changes.
*   **Data Validation:** Verify that all data points, budget figures, and project milestones match the live project management software (e.g., Jira, Asana, or MS Project).
*   **Stakeholder Review:** Route the draft through the designated project lead for accuracy and completeness before final conversion.

## Phase 2: PDF Conversion and Formatting
*   **Optimize for Export:** Use "Save as PDF" or "Print to PDF" features. Avoid "Print to Image" settings to ensure the text remains searchable and selectable.
*   **Embedding Fonts:** Ensure all fonts used in the source document are embedded in the PDF to maintain design consistency across different operating systems.
*   **Resolution Check:** Set output quality to "High Quality Print" (minimum 300 DPI) if the document contains images or technical diagrams.
*   **Accessibility Compliance:** Ensure the PDF is tagged for screen readers, including Alt-text for all charts, tables, and images.

## Phase 3: Final Review and Archiving
*   **Naming Convention:** Rename the file using the standardized naming convention: `YYYYMMDD_ProjectCode_DocumentName_v#.pdf`.
*   **Security Protocol:** Apply password protection or restricted editing permissions if the document contains PII (Personally Identifiable Information) or sensitive project financials.
*   **Metadata Entry:** Populate the PDF document properties (Title, Author, Subject, Keywords) within the file settings to assist in server-side indexing.
*   **Storage:** Upload the final file to the centralized Project Management Office (PMO) document repository (e.g., SharePoint, Google Drive, or local project server).

## Pro Tips & Pitfalls
*   **Pitfall - The "Screenshot Trap":** Never convert a document to PDF by taking a screenshot of the page. This destroys text-search functionality and reduces resolution.
*   **Pro Tip - Hyperlink Check:** Before exporting, test all internal and external hyperlinks. A PDF with broken links reflects poorly on project professionalism.
*   **Pitfall - Oversized Files:** If a PDF exceeds 10MB due to high-resolution photography, use an approved PDF compressor tool to reduce file size without losing readability.
*   **Pro Tip - Flattening Layers:** If your document contains complex layers (e.g., Photoshop or Illustrator imports), "flatten" the document before final PDF conversion to prevent rendering errors.

## Frequently Asked Questions (FAQ)

**1. Why is a standardized naming convention necessary?**
Standardized naming prevents "file rot," allowing team members to identify the project, the document type, and the version at a glance without having to open the file.

**2. Should I keep the source file (e.g., Word, Excel) after creating the PDF?**
Yes. Always retain the original source file in an "Editable" folder. The PDF is strictly for distribution and archival; the source file is required for future updates or revisions.

**3. What should I do if the PDF file is too large to email to stakeholders?**
Rather than emailing the file, upload the PDF to the central repository and send a secure link to the stakeholders. This keeps the document centralized and ensures everyone is viewing the most current version.
