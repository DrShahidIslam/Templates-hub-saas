# Standard Operating Procedure: Figma Onboarding Template Management

This SOP outlines the standardized process for deploying, maintaining, and customizing Figma onboarding templates to ensure a seamless integration experience for new hires. By utilizing a centralized, repeatable template, operations and design teams can reduce administrative overhead, ensure consistency in documentation, and accelerate the "time-to-productivity" for new team members. This procedure applies to all departmental onboarding flows managed within the Figma ecosystem.

## Phase 1: Preparation & Initialization
*   **Duplicate the Master Template:** Navigate to the "Onboarding Master Library" in Figma. Use the "Duplicate" function to create a unique copy for the specific new hire (format: `YYYY-MM-DD_Onboarding_[Name]`).
*   **Permissions Audit:** Ensure the new hire has been granted the appropriate Figma "Viewer" or "Editor" seat within the specific project team folder.
*   **Configure Access Levels:** Verify that the file permissions are set to "Anyone with the link can view" if sharing with external stakeholders, or "Restricted" for internal-only sensitive documentation.
*   **Stakeholder Sync:** Assign the "Onboarding Mentor" or Manager as a collaborator on the file with "Can Edit" access.

## Phase 2: Customization & Personalization
*   **Update Personal Metadata:** Update the cover slide with the new hire’s name, start date, and role title.
*   **Define Learning Tracks:** Modify the "Milestones" section to reflect the new hire’s specific 30-60-90 day goals.
*   **Link External Resources:** Hyperlink all external documentation (Notion, Slack channels, internal wikis) directly into the Figma frames to create a "Single Source of Truth."
*   **Initialize Checklists:** Populate the interactive checklists within the Figma file (use components with variants for "Pending" vs. "Completed" states).

## Phase 3: Deployment & Monitoring
*   **File Notification:** Send the direct link to the new hire via the primary communication channel (e.g., Slack/Email) with a brief orientation message.
*   **Walkthrough Session:** Schedule a 30-minute sync to walk the hire through the Figma file navigation, explaining how to mark tasks as complete and where to leave feedback.
*   **Progress Tracking:** Weekly review of the file—check for updated statuses, unresolved comments, or roadblocks flagged by the hire within the file’s "Parking Lot" or "Questions" section.

## Phase 4: Archiving & Iteration
*   **Completion Sign-off:** Once all milestones are reached, mark the file as "Archived" by moving it to the designated "Completed Onboarding" project folder.
*   **Feedback Loop:** Invite the new hire to leave a "Post-Mortem" comment on the onboarding experience within the Figma file for continuous improvement.
*   **Template Refinement:** Based on the feedback provided, update the "Master Template" to prevent common points of confusion for the next cohort.

## Pro Tips & Pitfalls
*   **Pro Tip:** Use Figma’s "Comments" feature as the primary asynchronous communication tool for onboarding questions; this keeps the context directly attached to the task.
*   **Pro Tip:** Use "Components" for recurring checklist items so that if you update a process, it updates across all current onboarding templates automatically.
*   **Pitfall:** Overloading the file with too many pages. Keep it to 5-7 core pages to avoid overwhelming the user.
*   **Pitfall:** Failing to check link permissions. Always verify that embedded links (e.g., Google Docs) have "Everyone at the company can edit" access, or you will create a bottleneck.

## Frequently Asked Questions

**Q: Should I use a separate Figma file for every new hire, or keep them all in one master file?**
A: Always duplicate a new file for each hire. Keeping them separate ensures data privacy, prevents clutter, and allows you to archive individual records without affecting the master template.

**Q: What if the new hire isn’t familiar with Figma?**
A: Include a "Figma Basics" slide or a link to a brief internal video tutorial on the first page of the onboarding template to remove the technical barrier to entry.

**Q: How often should the "Master Template" be updated?**
A: Conduct a formal review of the Master Template quarterly to ensure all links are active, processes are current, and branding reflects the latest company guidelines.
