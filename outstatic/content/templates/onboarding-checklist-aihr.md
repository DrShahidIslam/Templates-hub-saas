# Standard Operating Procedure: AIHR Onboarding Process

This Standard Operating Procedure (SOP) outlines the standardized process for onboarding new employees or team members onto the AIHR (Academy to Innovate HR) platform. Effective onboarding ensures that users gain immediate access to their designated certification tracks, understand the platform’s navigation, and are prepared to engage with the curriculum to maximize their professional development ROI. Following these steps ensures consistency, access security, and learner readiness.

## Phase 1: Administrative Provisioning and Access
*   **Verify User Licensing:** Confirm that the user has an active, valid license key allocated within the AIHR enterprise dashboard.
*   **Account Creation:** Initiate the invitation email via the AIHR admin console using the user’s corporate email address.
*   **Role Mapping:** Assign the appropriate program path (e.g., People Analytics, HR Digital Transformation) based on the user’s current role or development plan.
*   **Verification:** Ensure the user has received the activation link and successfully completed the password reset process.

## Phase 2: User Orientation and Platform Familiarization
*   **Initial Login:** Guide the user through the initial platform login to ensure there are no single sign-on (SSO) or browser compatibility issues.
*   **Profile Optimization:** Instruct the user to update their profile settings, including timezone settings and notifications, to ensure proper completion tracking.
*   **Platform Tour:** Provide a walkthrough of the "My Learning" dashboard, highlighting the navigation menu, progress bars, and the location of supporting resources (PDFs, tools, and templates).
*   **Community Integration:** Ensure the user has registered for the AIHR Community forum to facilitate peer-to-peer learning and networking.

## Phase 3: Alignment and Goal Setting
*   **Establish Completion Timeline:** Set a target date for the certification completion with the user to ensure accountability.
*   **Identify Integration Points:** Discuss how the specific AIHR modules align with the employee’s current OKRs (Objectives and Key Results).
*   **Resource Allocation:** Confirm that the user has dedicated "learning hours" blocked off in their calendar to ensure consistent progress.

## Pro Tips & Pitfalls
*   **Pro Tip:** Encourage the user to download the AIHR mobile app if available; it helps with consuming bite-sized video content during commutes or downtimes.
*   **Pro Tip:** Recommend that the user creates a dedicated "AIHR Notes" folder in their professional workspace to store templates and frameworks for immediate application.
*   **Pitfall:** Avoid "information overload" by not forcing the user to explore every elective module on Day 1. Focus strictly on their assigned core path.
*   **Pitfall:** Failure to check browser cache or ad-blocker settings often leads to video playback errors; ensure the user's browser is optimized for the platform before they start their first lesson.

## Frequently Asked Questions (FAQ)

**1. What should I do if the user does not receive the AIHR activation email?**
First, verify the email address was entered correctly in the admin console. Advise the user to check their spam or junk folders, and if it remains missing, manually trigger a password reset link from the AIHR login page.

**2. How do we handle license reassignment if an employee leaves the team?**
Access the "Admin" settings in the AIHR portal, deactivate the current user’s account, and then remove their license assignment to make it available for the incoming team member. Note that progress data for the departing user may be archived depending on your subscription level.

**3. Are there prerequisites required before starting a specific AIHR track?**
While most AIHR tracks are designed to be standalone, it is recommended to review the "Course Overview" document provided on the landing page of each track to ensure the user has the foundational knowledge required for advanced data or technical modules.
