# Standard Operating Procedure: Creation and Management of Teacher PDFs

## Introduction
This Standard Operating Procedure (SOP) outlines the professional requirements and technical standards for generating, formatting, and distributing instructional PDFs for teachers. The objective is to ensure that all digital learning materials are accessible, visually consistent, brand-aligned, and optimized for both screen reading and physical printing. By following these guidelines, educators can minimize technical issues for students and ensure a seamless learning experience across various device platforms.

## Step-by-Step Checklist

## Phase 1: Document Structure & Formatting
*   **Establish Hierarchy:** Use Heading 1 (H1) for the main title, H2 for major sections, and H3 for sub-sections. Do not use bold text as a substitute for headings.
*   **Standardize Typography:** Use high-legibility sans-serif fonts (e.g., Arial, Calibri, or Open Sans) at a minimum size of 11pt for body text.
*   **Consistent Layout:** Apply uniform margins (1-inch recommended) and page numbering in the footer.
*   **Color Contrast:** Ensure a minimum 4.5:1 contrast ratio between text and background for accessibility (WCAG compliance).

## Phase 2: Technical Optimization
*   **Hyperlink Verification:** Ensure all external URLs are active. Use descriptive text for links (e.g., instead of "click here," use "View the Periodic Table PDF").
*   **Image Optimization:** Compress all embedded images to keep the file size under 5MB to ensure quick loading for students with limited bandwidth.
*   **Flattening Layers:** Flatten all complex graphics or layered elements to prevent formatting shifts when opening the file on different operating systems.
*   **Security Settings:** If distributing sensitive assessment materials, apply document-level password protection or "Read Only" permissions.

## Phase 3: Accessibility & Final Review
*   **Alternative Text:** Add Alt-Text to all images and charts to ensure compatibility with screen readers.
*   **Tagging:** Use the "Autotag Document" feature in Adobe Acrobat to ensure logical reading order for assistive technology.
*   **Spell Check/Proofreading:** Conduct a final pass for instructional clarity and grammatical accuracy.
*   **Test Print:** Print a hard copy of the document to ensure no content is cropped by printer margins.

## Pro Tips & Pitfalls
*   **Pro Tip:** Always save the original editable source file (e.g., Word or Canva) separately. Never delete the source file after exporting the PDF.
*   **Pro Tip:** Use the "Save as PDF" function rather than "Print to PDF" whenever possible to preserve active hyperlinks and document metadata.
*   **Pitfall:** Avoid embedding large video files directly into a PDF, as this can cause file corruption or extreme lag. Instead, link to a hosted video service like YouTube or Vimeo.
*   **Pitfall:** Do not use low-resolution screenshots for diagrams. If text is blurry at 200% zoom, the file is not professional-grade.

## Frequently Asked Questions (FAQ)

**Q: Should I use portrait or landscape orientation?**
A: Use portrait for text-heavy documents like reading assignments or instructions. Use landscape only for specialized diagrams, charts, or presentations intended specifically for interactive whiteboards.

**Q: Why does my PDF look different on a student’s phone compared to my laptop?**
A: This usually happens if you used non-standard fonts. Always "embed fonts" when saving/exporting to PDF to ensure the document appears identical on all devices.

**Q: How do I handle large file sizes for school portals?**
A: Use an online PDF compressor or the "Reduce File Size" option in Adobe Acrobat to strip unnecessary metadata and downsample images without sacrificing legibility.
